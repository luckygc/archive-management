## Why

当前登录能力只提供当前主体查询和退出登录，缺少管理员查看登录会话、踢下线和审计登录/退出/踢出事件的能力。认证排障和安全审计需要保留登录尝试、客户端信息、操作人和会话处理结果。

## What Changes

- 新增登录会话列表接口，以 Spring Session 当前有效会话作为在线真相源。
- 新增登录会话踢下线接口，直接删除目标 Spring Session。
- 新增认证审计日志，记录登录成功、登录失败、主动退出和管理员踢下线事件。
- 新增登录失败限制状态，按用户名统计连续失败并执行指数退避锁定。
- 新增短生命周期数据统一清理接口和 Quartz 作业，CAP challenge/token 与登录失败限制接入该清理机制。
- 登录成功时保存客户端上下文到 Spring Session 属性，并写入认证审计日志。
- 认证事件记录客户端 IP、Host、Forwarded 相关头、原始 User-Agent 和后端解析出的浏览器、操作系统、设备类型等摘要。
- PC 端新增“登录会话”和“认证审计”页面，用于查看在线会话、踢下线和查询审计记录。

## Impact

- 后端认证模块新增认证审计表、实体/Repository、服务、接口 DTO 和控制器方法。
- 后端认证模块新增登录失败限制表、实体/Repository 和限制服务。
- 后端新增短生命周期数据清理接口与 Quartz 作业，避免 CAP 在创建、兑换、校验请求中顺手清理过期数据。
- Spring Security 登录成功、登录失败、退出登录处理链路需要写入认证审计。
- Spring Session JDBC 表继续作为在线会话真相源，不新增项目自有登录会话表。
- 前端 `frontend-core` 新增登录会话和认证审计 API 类型与调用方法。
- Web 端新增系统配置下的“登录会话”和“认证审计”页面。
