# 用户手册

本文面向档案管理人员、业务经办人和系统管理员，说明 PC 主应用的页面地图和常见操作路径。界面以左侧菜单、多页签、表格、筛选、抽屉和弹窗为主要交互方式。

## 登录

1. 打开 PC 主应用。
2. 在登录页完成 CAP 安全验证。
3. 输入账号和密码。
4. 登录成功后进入工作台。

登录失败次数过多时，账号会受到失败限制。管理员可在认证审计和登录失败限制相关能力中排查。

## 基础配置流程

首次配置档案业务时，建议按以下顺序：

1. 在组织架构中维护部门。
2. 在用户管理中维护用户并绑定部门。
3. 在角色管理中维护角色。
4. 在授权管理中维护功能权限。
5. 在全宗管理中创建全宗。
6. 在档案分类中创建分类方案、分类、字段、布局和唯一规则。
7. 为分类构建动态表。
8. 在治理方案中维护治理版本、适用范围和绑定。
9. 在数据范围中维护档案可见范围，并绑定到角色、用户或部门。

## 档案管理流程

常见档案条目维护路径：

1. 进入档案管理。
2. 选择全宗和档案分类。
3. 新增档案条目并填写固定字段和动态字段。
4. 上传电子文件。
5. 需要成卷时创建案卷，并将条目加入案卷。
6. 需要限制修改时锁定档案条目。
7. 后续修改、删除、导出或下载文件时，由后端按权限和数据范围再次校验。

档案分类动态字段由分类字段定义和布局控制。字段发生变化后，需要确认动态表和搜索投影已经按页面操作完成构建或重建。

## 档案搜索流程

档案搜索面向查询和发现：

- 管理查询用于结构化筛选、排序和分页。
- 全文发现用于普通用户基于关键词查找档案。
- 数据范围会限制用户可见结果。
- 查询结果中的下载和查看动作仍会再次校验权限。

如果查询结果少于预期，优先确认全宗、分类、数据范围、字段是否可检索，以及全文检索 provider 是否可用。

## 导入导出流程

导入：

1. 在档案管理或分类相关入口下载只有表头的空白导入模板。
2. 按模板填写档案字段。
3. 上传 Excel 文件。
4. 查看导入结果和错误回填。
5. 对失败行修正后重新导入。

导出：

1. 在档案管理或档案搜索中设置筛选条件。
2. 发起导出。
3. 系统按当前用户的数据范围和权限生成 Excel。

导入导出不绕过服务端校验。导入时唯一规则、字段类型、必填、全宗和分类等规则都会影响结果。

## 电子文件流程

档案电子文件挂在具体档案条目下。

常见操作：

- 查看电子文件列表。
- 上传文件并设置用途类型和显示顺序。
- 删除电子文件。
- 创建下载短链。
- 通过短链下载文件。

公开短链和登录短链使用不同路径。公开短链只用于业务明确允许公开访问的场景。

## 治理、本体和规则

治理方案用于表达某个机构、客户或场景采用的一组档案本体、规则和配置引用。

本体管理用于维护档案对象、属性、关系、事件和字段映射。

本地规则用于维护校验、提示、阻断等规则，并通过发布、启用、禁用控制运行状态。规则追踪用于排查规则为什么命中或未命中。

建议先在测试环境验证治理方案和规则，再用于正式档案数据。

## 审批流程

流程管理员使用“流程定义”维护可视化审批流程：

1. 新建流程并填写唯一编码、名称和业务类型，画布会自动创建开始、审批和结束节点。
2. 将审批节点或条件分支拖入画布，通过锚点连线；审批节点选择指定候选用户，条件分支配置一个默认分支和受控字段条件。
3. 使用画布工具撤销、重做、缩放或适应画布；右侧“发布检查”会列出断路、缺少办理人和条件不完整等问题，点击问题可定位元素。
4. 保存草稿后发布版本；后续修改重新发布会产生不可变新版本，运行中的实例继续使用启动版本。
5. 需要停止新发起时停用定义，已有实例不受影响。

具备发起权限的用户可在“审批中心”发起通用审批。“我的待办”和“已办”来自统一待办投影，可汇总审批及后续接入的其他业务；候选用户填写意见并同意或驳回，同意后按流程图进入下一节点或条件分支，驳回会直接终止流程。发起人可在流程结束前撤回。

最小人工验证：

1. 创建包含两个指定用户节点的定义并发布。
2. 发起实例，确认“我发起的”显示第一个当前节点。
3. 使用第一节点候选用户同意，确认待办转到第二节点且历史保留第一条意见。
4. 使用第二节点候选用户同意，确认实例状态为“已通过”。
5. 另发起一个实例并驳回，确认实例状态为“已驳回”且没有未完成待办。
6. 另发起一个实例并由发起人撤回，确认当前待办关闭且实例状态为“已撤回”。

## 权限和数据范围

权限分两层：

- 功能权限：决定用户能不能进入页面或执行动作。
- 数据范围：决定用户能看见或操作哪些档案集合。

遇到“看不到数据”或“按钮不可用”时，按以下顺序排查：

1. 用户是否启用。
2. 用户是否绑定正确角色。
3. 角色是否启用并具备功能权限。
4. 用户、角色或部门是否绑定数据范围。
5. 数据范围条件是否覆盖目标全宗、分类和动态字段。

## 系统管理

系统管理员常用入口：

- 用户管理：创建用户、更新用户、重置密码、绑定角色。
- 角色管理：创建角色、维护角色状态。
- 授权管理：维护权限点和授权关系。
- 数据范围：维护档案可见范围。
- 登录会话：查看在线会话并踢下线。
- 认证审计：排查登录失败、退出和踢下线事件。

运行配置以部署配置和后端配置文件为准，PC 主应用不提供运行配置修改入口。
