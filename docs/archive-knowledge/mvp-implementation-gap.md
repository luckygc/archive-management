# 档案系统 MVP 实施缺口

本文把 [档案系统能力地图](./archive-system-capability-map.md) 中的 MVP 目标落成可执行缺口清单。它不是接口合同；涉及字段、状态、接口、验收的内容以 OpenSpec 为准。

## MVP 目标

MVP 需要让早期用户持续维护真实档案，而不是只完成演示链路。判断口径是：

- 一个部门可以维护全宗、分类、字段、案卷、卷内条目和文件。
- 一批真实档案可以导入、校验、查询、导出。
- 权限和数据范围不依赖前端隐藏按钮兜底。
- 文件下载基于文件记录路由，不靠全局配置猜历史位置。
- 关键写入和下载导出动作有可追溯审计。

## 当前已具备

| 能力                     | 状态         | 依据                                                                     |
| ------------------------ | ------------ | ------------------------------------------------------------------------ |
| 分类树、动态字段、动态表 | 已有核心基础 | `archive-metadata` 规格覆盖分类、字段、布局、唯一规则和动态表            |
| item/volume 分对象管理   | 已有核心基础 | `archive-metadata` 已定义 `ArchiveItem`、`ArchiveVolume`、关联和明细行表 |
| 管理查询和全文发现       | 已有核心基础 | `archive-record-search` 已定义结构化筛选、排序、全文 provider 和投影     |
| 文件记录和存储路由       | 已有规格基础 | `file-storage` 已定义本地文件记录、对象键、存储路由和 SHA-256            |
| 登录和基础角色           | 已有基础     | `login-authentication` 与授权表已存在，细粒度数据范围仍需补齐            |
| Intake 入口              | 仅有入口     | `intake` 当前只是模块边界，不等于导入、移交或接收流程                    |

## MVP 前必须补齐

| 缺口               | MVP 口径                                                  | 不纳入本阶段                       |
| ------------------ | --------------------------------------------------------- | ---------------------------------- |
| MVP 范围门槛       | 明确哪些能力完成后才能称为 MVP                            | 完整平台路线、长期保存全生命周期   |
| 全宗基础治理       | 多全宗编码、名称、启停、排序；业务写入必须引用启用全宗    | 全宗沿革、全宗关系、全宗卷         |
| 手工档号和唯一校验 | 保存 `archive_no`，通过分类唯一规则约束手工档号或组合档号 | 档号规则引擎、流水、补号和档号历史 |
| 基础导入导出       | 至少支持标准 Excel 批量导入、校验错误回填、按查询结果导出 | 标准移交包、跨系统重传、馆际交换   |
| 基础审计           | 创建、修改、删除、锁定、解锁、下载、导出的关键审计        | 证据包、签名、时间戳、不可抵赖链   |
| 基础权限和数据范围 | 后端按登录、接口权限、全宗/分类范围限制查询和写入         | 复杂密级、开放控制、利用审批       |
| 档案电子文件管理   | 档案条目可上传、查看列表、下载和删除电子文件，列表可见和下载权限分离；在线预览暂不纳入本阶段 | 主文件/附件/组件/派生件完整模型    |

## 建议实施切片

### 第一片：MVP 合同与门槛

目标是固定 MVP 边界，防止后续把 POC 已完成误判为 MVP 已完成。本轮 OpenSpec change `add-archive-mvp-foundation` 负责补齐该合同。

交付物：

- MVP 必备能力清单。
- 已有能力与缺口映射。
- 后续代码任务分片。

### 第二片：全宗、档号和基础审计收口

目标是让档案写入可归属、可唯一、可追踪。

交付物：

- 全宗启停校验。
- `archive_no` 手工录入和分类唯一规则组合。
- item/volume 写入、删除、锁定、解锁审计。

### 第三片：档案电子文件、下载审计和导入导出

目标是支持真实批量维护和文件利用。

交付物：

- 档案条目上传和维护电子文件。
- 下载动作按文件记录路由并写审计。
- Excel 导入、错误回填和查询结果导出。

### 第四片：数据范围最小实现

目标是权限不靠前端兜底。

交付物：

- 后端查询和写入按用户可见全宗、分类范围过滤。
- 管理查询、全文发现和详情读取共享同一范围判断。
- 越权写入返回标准错误。

## 推迟到基础能力

- 分类方案版本、历史冻结和迁移。
- 完整档号规则引擎、流水和历史。
- 全宗沿革、关系和立档单位治理。
- 保管期限表、归档范围和到期处置。
- 正式归档流程、移交接收、四性检测和利用审批。
- 长期保存格式迁移、证据包和备份恢复验收。
