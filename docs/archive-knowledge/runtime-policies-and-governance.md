# 运行时约束、规则与治理方案

本文记录 0.0.1 的当前设计基线。可执行合同以 `openspec/specs/` 为准。

## 核心模型

治理方案版本直接拥有用户定义的运行时约束和规则。不同机构继续使用自己的分类、固定字段和动态字段；系统在运行时按治理版本、分类和触发点生成带类型的真实字段目录，不维护第二套字段解释或映射资源。

用户可以定义：

- 作用域：治理版本、全宗、分类和固定触发点。
- 条件：只允许 `all`、`any`、`not`、类型化比较和空值判断组成的结构化 AST。
- 约束：断言失败后执行 `REJECT` 或 `WARN`。
- 规则：条件命中后按顺序执行系统注册的 `REJECT`、`WARN` 或 `SET_FIELD`。

用户不能提交 SQL、脚本、类名、Bean 名、函数名或任意副作用实现。

## 固定执行边界

首批固定触发点为：

- `ITEM_BEFORE_CREATE`
- `ITEM_BEFORE_UPDATE`
- `ITEM_BEFORE_DELETE`
- `VOLUME_BEFORE_CREATE`
- `VOLUME_BEFORE_ADD_ITEM`
- `FILE_BEFORE_UPLOAD`
- `EXPORT_BEFORE_CREATE`

条目、案卷、电子文件、导入和导出 Service 在持久化或生成文件前调用同一执行核心。执行顺序为 `priority ASC, definition_code ASC, id ASC`；规则先修改尚未持久化的候选值，约束再检查最终候选值。阻断、配置失效、赋值冲突或执行器错误均失败关闭，并由业务事务保证主数据、动态表、审计和决策追踪不产生部分写入。

动作处理器只返回候选值变化或决策，不直接调用 Repository、Mapper、流程、事件、远程服务或文件系统。服务端校验、权限、数据范围和 PostgreSQL 约束仍是最终边界。

## 发布与追踪

定义先保存为 `DRAFT`，发布时完整校验作用域、真实字段、类型、AST、触发点和动作兼容性。已发布定义及动作不可原地修改或删除，只能通过受审计的启用、停用操作改变运行态开关。

真实业务执行成功后保存最小决策摘要。追踪查询先在 PostgreSQL 中应用功能权限和档案数据范围，再按统一 cursor 合同分页；追踪不保存未脱敏的候选字段全文。试运行复用同一执行核心，但不写主数据、文件、审计或追踪。

## 动态配置迁移与恢复

运行时配置快照用于快速备份和跨环境迁移动态部分，不替代数据库整库备份、WAL/PITR 或对象存储恢复。

- 导出使用治理方案、版本、分类和字段稳定编码，不包含数据库 ID、档案数据、文件、用户、权限、审计或追踪。
- 规范化内容带格式版本和 SHA-256 摘要；重复导出相同配置得到稳定摘要。
- 导入前完整校验摘要、大小、节点数、目标分类、字段类型、读写能力、触发点和动作兼容性。
- 跨环境导入只创建新的 `DRAFT` 治理版本，不覆盖或自动发布现有版本。
- 恢复只允许管理员明确选择 `DRAFT` 目标，并在单个事务中全量替换运行时定义；任一步骤失败时旧草稿保持不变。

## 存储原则

档案实例仍保存在固定主表、分类动态表、明细表、文件组件表和关系表。运行时配置使用受控 `jsonb` 保存条件和动作参数，但主数据不进入万能 JSON 或 EAV。数据库迁移只面向 PostgreSQL，发布不可变性和跨表状态不变量由命名约束及触发器兜底。
